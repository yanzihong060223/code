 # C++ Memory Pool

  一个基于 C++ 实现的轻量级内存池项目，用于优化频繁小对象申请与释放场景下的性能问题。项目通过预申请大块内存、按固定大小切分内存块、空闲
  链表复用等方式，减少系统调用和堆管理开销，提高内存分配效率。

  ## 项目特点

  - 使用内存池管理小块内存，降低频繁 `new` / `delete` 或 `malloc` / `free` 带来的性能损耗
  - 通过空闲链表维护可复用内存块，实现快速分配与回收
  - 支持内存块复用，减少内存碎片
  - 封装统一的申请和释放接口，便于业务代码接入
  - 代码结构清晰，适合作为 C++ 内存管理、底层系统编程学习项目

  ## 项目背景

  在高并发服务、对象池、网络服务器、缓存系统等场景中，程序会频繁申请和释放大量小对象。如果每次都直接调用系统堆分配器，可能会带来以下问
  题：

  - 分配和释放成本较高
  - 容易产生内存碎片
  - 多线程场景下可能存在锁竞争
  - 小对象分配效率不稳定

  内存池的核心思想是：一次性向系统申请较大的连续内存，再由程序自己管理这些内存块。后续对象申请时优先从内存池中获取，释放时归还到内存
  池，避免频繁访问系统堆。

  ## 核心设计

  ### 1. 内存块管理

  内存池内部将大块内存切分成多个固定大小的小块，每个小块用于满足一次内存申请。

  ### 2. 空闲链表

  释放后的内存块不会立即归还给系统，而是挂回空闲链表中。下一次申请相同大小的内存时，可以直接从空闲链表取出，提高复用率。

  ### 3. 批量申请

  当空闲链表为空时，内存池会重新向系统申请一大块内存，并切分成多个小块加入管理结构中，减少系统调用次数。
